# Curator Bookmarks 隐私政策

最后更新：2026-04-30

Curator Bookmarks 是一个本地优先的 Chrome 书签管理扩展。我们重视用户隐私，并尽量将书签管理相关数据保存在用户自己的浏览器本地。

## 单一用途

Curator Bookmarks 的用途是帮助用户管理、整理、搜索、清理和展示 Chrome 书签，包括弹窗书签管理、书签新标签页、书签可用性检测、重复书签清理，以及用户主动启用时的 AI 辅助整理。

## 我们处理哪些数据

扩展可能处理以下类型的数据：

- **书签数据**：书签标题、URL、文件夹结构、书签位置和排序。
- **网站内容与搜索查询**：在用户主动使用 AI 命名、智能分类、自动分析、Popup 自然语言搜索或可选远程解析时，扩展可能读取当前网页或书签目标网页的标题、URL、元信息、页面文本摘要，或处理用户输入的搜索查询。
- **认证信息**：用户填写的 AI API Key。
- **扩展设置与整理记录**：检测历史、忽略规则、回收站记录、重定向缓存、新标签页设置、自定义图标和背景设置。

## 数据如何保存

Curator Bookmarks 默认将数据保存在本地：

- 书签整理设置、检测历史、忽略规则、回收站、Popup 偏好和 AI 设置保存在 `chrome.storage.local`。
- 新标签页背景、本地媒体缓存和图片 URL 缓存可能保存在浏览器本地的 IndexedDB 中。
- 用户填写的 AI API Key 仅保存在当前扩展的本地存储中。

我们没有自己的服务器，也不会把用户书签库上传到开发者服务器。

## 数据如何使用

扩展仅将数据用于以下功能：

- 搜索、浏览、编辑、移动、创建、排序和删除 Chrome 书签。
- 在新标签页中展示用户指定书签文件夹中的书签。
- 检测书签链接是否可访问，以及识别可能的重定向地址。
- 查找重复书签并辅助清理。
- 在用户主动配置并启用 AI 功能时，为书签生成标题建议、推荐保存文件夹或改写 Popup 自然语言搜索查询。
- 保存用户的设置、历史记录和回收站数据，以便后续查看或恢复。

## 第三方服务

Curator Bookmarks 的基础书签管理功能不需要第三方服务。

当用户主动配置并使用 AI 功能时，扩展会向用户设置的 OpenAI 兼容 AI 服务发送必要请求。请求内容可能包含书签标题、URL、网页元信息、页面文本摘要，或用户在 Popup 自然语言搜索中输入的查询文本。用户的 API Key 会作为认证信息发送给用户配置的 AI 服务提供方。

当用户主动启用 Jina Reader 远程解析时，扩展会将目标网页 URL 发送给 Jina Reader，并使用其返回的页面文本辅助 AI 整理。

这些第三方服务的数据处理方式由对应服务提供方的隐私政策约束。请不要将隐私敏感页面用于远程解析或 AI 整理。

## 数据分享与出售

我们不会出售用户数据。

我们不会将用户数据用于与扩展单一用途无关的目的。

除用户主动配置并使用 AI 服务或 Jina Reader 远程解析所需的请求外，我们不会将用户数据转让给第三方。

我们不会使用用户数据判断信用、贷款资格或类似金融用途。

## 权限用途

扩展使用的 Chrome 权限仅用于实现书签管理功能：

- `bookmarks`：读取、创建、编辑、移动、排序和删除书签。
- `storage`：保存扩展设置、整理历史、忽略规则、回收站和新标签页设置。
- `activeTab`：在用户主动操作时读取当前标签页信息，用于收藏当前网页或智能分类。
- `favicon`：显示书签网站图标。
- `notifications`：在后台整理或自动分析完成时通知用户。
- `alarms`：在 MV3 后台服务被浏览器回收后继续处理用户已启用的任务。
- `webNavigation` / `webRequest`：用于书签可用性检测和重定向判断。
- `optional_host_permissions`：在用户主动检测网页、读取网页内容或使用 AI 服务时，按需请求对应站点访问权限。

## 远程代码

Curator Bookmarks 不加载或执行远程 JavaScript / Wasm 代码。扩展运行代码随扩展包一起发布。

向 AI 服务、Jina Reader 或网页 URL 发送请求仅用于获取数据或服务响应，不用于加载并执行远程代码。

## 用户控制

用户可以通过以下方式控制数据：

- 在扩展设置页关闭或调整相关功能。
- 删除检测历史、回收站记录、忽略规则或新标签页自定义内容。
- 删除或更换 AI API Key。
- 在 Chrome 扩展管理页面移除 Curator Bookmarks，以删除扩展本地数据。
- 在 Chrome 权限设置中撤销已授予的站点访问权限。
