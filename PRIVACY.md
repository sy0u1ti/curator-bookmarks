# Curator Bookmarks 隐私政策

最后更新：2026-05-08

Curator Bookmarks 是本地优先的 Chrome 书签搜索、整理、清理和新标签页扩展；AI 与远程解析为用户主动启用的可选辅助能力。

## 单一用途

Curator 的用途是管理用户自己的 Chrome 书签：Popup 搜索、Dashboard 批量整理、新标签页书签入口、重复/死链治理、回收站、备份恢复，以及用户主动启用时的 AI/Jina 辅助整理。

Curator 不提供广告、推荐流、账号云同步或默认远程行为遥测。这里的“本地优先”表示核心书签读取、搜索和整理默认在本机执行；它不表示所有功能永远离线。

## 我们处理哪些数据

- 书签数据：标题、URL、文件夹结构、位置、排序和创建/修改信息。
- 标签与摘要：手动标签、AI 标签、摘要、内容类型、别名和整理记录。
- 设置与本地记录：新标签页设置、Speed Dial、最近通过 Curator 打开的书签记录、检测历史、忽略规则、重定向缓存、回收站、备份元数据。
- 可选 AI/Jina 数据：用户启用后，可能处理标题、URL、文件夹路径、标签、摘要、网页正文片段和用户查询。
- AI 用量计数：本地保存当天 AI 请求次数，用于执行每日上限；不上传。
- 认证信息：用户填写的 AI API Key 或 Authorization 所需字段。
- 外部请求审计摘要：时间、功能类型、目标域名、状态、字段类别和条数。

## 数据如何保存

- Chrome 书签仍保存在 Chrome 原生书签系统中。
- 扩展设置、标签、检测历史、回收站、忽略规则、AI 用量计数和脱敏审计摘要保存在本地扩展存储。
- 大型本地数据、内容索引或新标签页媒体缓存可能保存在本地 IndexedDB。
- AI API Key 仅保存在本地扩展存储中，不进入普通备份、审计日志、截图或错误报告。

普通备份默认排除 API Key、Authorization、外部请求审计日志、AI 用量计数、网页正文/摘要缓存、用户媒体缓存和敏感诊断日志。

## 外部请求边界

Curator 没有开发者自有服务器，不会把完整书签库上传到开发者服务器。以下功能可能离开设备：

| 类别 | 默认状态 | 触发条件 | 发送或暴露内容 | 接收方 | 用户控制 |
| --- | --- | --- | --- | --- | --- |
| 本地书签搜索 | 开启 | 用户输入搜索 | 不离开设备 | 无 | 核心能力 |
| Newtab 网页搜索 | 可关闭 | 用户提交网页搜索 | 搜索关键词和浏览器常规网络请求元数据 | 所选搜索引擎，例如 Google、Bing、百度、DuckDuckGo | 在新标签页设置关闭“启用网页搜索” |
| 精选远程背景 | 默认关闭 | 用户主动选择精选图库 | 图片请求、IP、User-Agent 等常规网络元数据；不包含书签数据 | NASA、The Met、Smithsonian、Rijksmuseum 或 CDN | 切换为纯色、本地媒体或删除远程背景 |
| 用户自定义远程背景 | 默认关闭 | 用户输入并保存图片 URL | 图片请求和网络元数据；不包含书签数据 | 用户配置的服务方 | 删除 URL 或切换背景类型 |
| 链接可用性/重定向检测 | 默认关闭 | 用户主动运行检测 | 被检测 URL 请求、HTTP 状态、重定向证据；日志默认只保留域名和状态类别 | 目标网站 | 选择范围、暂停、取消、清空历史 |
| Jina Reader 远程解析 | 默认关闭 | 用户启用并启动解析/AI 分析 | 目标 URL 和 Jina 返回内容 | `https://r.jina.ai/` | 关闭 Jina、跳过敏感 URL、清缓存 |
| AI 服务 | 默认关闭 | 用户配置 provider/API Key 并主动发起 | 标题、URL、文件夹、标签、摘要、正文片段、用户查询等字段类别 | 用户配置的 AI 服务 | 关闭 AI、删除 API Key、预览字段、停止批次 |

远程背景不作为主机权限理由。远程 SVG 背景会被拒绝缓存或安全处理。

AI 请求发送前会在 Options 展示目标 Base URL、字段类别、预计条数、每批/每日上限和正文片段策略。网页内容和用户查询会被作为不可信输入处理；提示词要求模型不得执行网页内容中的指令。

## 权限用途

- `bookmarks`：读取、创建、编辑、移动、排序和删除 Chrome 书签。
- `storage`：保存扩展设置、标签索引、检测历史、回收站、脱敏审计摘要和新标签页配置。
- `activeTab`：用户主动保存或分析当前页面时读取当前标签页标题/URL。
- `favicon`：显示书签网站图标。
- `notifications`：后台任务完成、失败或停止时显示本地通知。
- `alarms`：Manifest V3 service worker 休眠后继续处理用户已启用的任务。
- `webNavigation` / `webRequest`：仅用于用户主动运行的链接可用性检测、重定向证据和后台检查标签页状态，不读取全量浏览历史。
- `optional_host_permissions` `http://*/*` 与 `https://*/*`：当前版本按需请求，用于用户触发的任意书签 URL 链接检测、可选内容提取、用户配置 AI 服务请求和可选 Jina 请求。安装时不授予全站点访问；Newtab 网页搜索和远程背景不是该权限理由。
- `chrome_url_overrides.newtab`：Curator 会替换 Chrome 新标签页，用于书签搜索和快捷入口。

权限决策矩阵见 `docs/permissions-matrix.md`。

## 审计日志与诊断

外部请求审计日志默认只保存最近 20 条或 7 天内的脱敏摘要。它只记录时间、功能类型、目标域名、状态、字段类别和条数。

审计日志默认不保存完整 URL query、请求正文、API Key、Authorization header、网页正文、AI 完整输入输出。普通备份不包含审计日志；只有用户手动选择并确认包含诊断信息时才应导出。

## 安全整理

删除默认进入 Curator 回收站或保留恢复记录。永久删除、清空回收站、批量移动/删除、批量标签修改、备份恢复、应用 AI 建议和启用计划任务等高风险操作应提供预览、确认、恢复点或回滚说明、部分失败处理和结果摘要。

AI 不会在未预览和确认的情况下自动永久删除或覆盖用户手动整理结果。

## 数据分享与出售

我们不会出售用户数据，不会把用户数据用于广告画像、信用、贷款资格或与扩展单一用途无关的用途。

除用户主动触发的网页搜索、远程背景、链接检测、Jina 或用户配置 AI 服务外，Curator 不会把书签数据发送给第三方。

## 远程代码

Curator 不加载或执行远程 JavaScript / Wasm 代码。扩展运行代码随扩展包发布。AI/Jina/搜索引擎/图片响应只作为数据或媒体处理，不作为代码执行。

## 用户控制

- 在新标签页设置关闭网页搜索、精选图库或自定义远程背景。
- 在 Options 关闭 AI/Jina、删除 API Key、停止批量任务。
- 清空检测历史、回收站、忽略规则、审计摘要或本地缓存。
- 导出备份前查看数据类别；恢复前查看预览和警告。
- 在 Chrome 扩展管理页面停用或移除 Curator 以恢复 Chrome 默认新标签页。
